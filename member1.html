1. What Traffic Management Means in Our Project

Problem definition & graph view of the city

Our city faces typical issues: long queues at peak hours, random pedestrian crossing, and signals that do not respect real-time traffic load. To analyse this in a structured way, we convert the road network into a weighted graph.

  • Intersections → graph nodes
  • Road segments → edges with weights
  • Weight → travel time, distance, or congestion level

On top of this graph, we apply algorithms like Dijkstra, BFS, DFS, MST, and classical sorting to solve real traffic sub-problems: routing, signal control, sensor placement, and data reporting.

Goal of the project

Show how Data Structures & Algorithms are not only theory– but actually match real traffic behaviours:

  • Graphs → city roads
  • Queues → vehicles waiting at signals
  • Scheduling → signal timing
  • Shortest path → route suggestion apps
  • MST → cheap sensor/camera network

2. Smart Route Planning for Vehicles

Finding fast and less congested paths

When drivers choose a wrong route, they not only delay themselves but also push more vehicles into already jammed roads. So, the first part of traffic management is good route planning.

Problem: Suggest the best route between any two junctions considering time and congestion.

Algorithms used
  • Dijkstra – to compute the fastest path using current travel times.
  • Bellman–Ford – to handle dynamic penalties (e.g., accident, road work).
  • Floyd–Warshall – precompute all-pairs shortest paths for emergency vehicles.

These algorithms allow a routing system to quickly adjust when the weight (travel time) changes due to rain, accident, or a sudden jam.

  • Balanced use of parallel roads instead of overloading one route.
  • Lower total travel time and fuel usage.
  • Emergency routes can be reserved and optimised.
Pseudocode: Dijkstra for travel time
function dijkstra(graph, source):
    for each node v:
        dist[v] = INF
    dist[source] = 0
    PQ = min-priority-queue()
    PQ.push(source, 0)

    while PQ not empty:
        (u, cost) = PQ.pop_min()

        if cost > dist[u]:
            continue     // outdated entry

        for each edge (u, v, w) in graph[u]:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                PQ.push(v, dist[v])

    return dist
Correctness: non-negative weights DS used: adjacency list + min heap

3. Adaptive Traffic Signal Control

Giving more green time where traffic is higher

Fixed-time signals waste precious seconds when one road is empty but still getting green, while another side is overflowing with vehicles. We aim for adaptive signal timing.

Problem: Distribute a fixed signal cycle time fairly based on vehicle count.

Algorithms & data structures
  • Queue – stores vehicles waiting at each direction.
  • AVL Tree / Balanced BST – keeps junctions sorted by congestion.
  • Heap (priority queue) – quickly picks the most congested approach.
  • Greedy scheduling – divide cycle time in proportion to demand.

Instead of equal green for all, our scheduler allocates more green to approaches with more queued vehicles.

Pseudocode: Greedy green-time split
function computeGreenTimes(count[1..n], cycleTime):
    total = 0
    for i = 1 to n:
        total += count[i]

    for i = 1 to n:
        share[i] = (count[i] / total) * cycleTime

    return share
Mini C snippet: queue at one junction
#include <stdio.h>

int main() {
    int queue = 0, in, out;

    for (int cycle = 1; cycle <= 5; cycle++) {
        printf("\nCycle %d\n", cycle);
        printf("Vehicles arrived: ");
        scanf("%d", &in);
        printf("Vehicles passed on green: ");
        scanf("%d", &out);

        if (queue < 0) queue = 0;

}

4. Handling Accidents & Road Closures

Keeping the network connected when something fails

When an accident blocks a road, some areas may become very hard to reach. The traffic control centre must quickly know which regions are affected, and how to reroute vehicles.

Problem: After removing a road, check connectivity and suggest alternate routes.

Algorithms used
  • DFS / BFS – explore reachable zones from a source junction.
  • Disjoint Set (Union–Find) – maintain components when roads go down.
  • Bellman–Ford – recompute routes with extra delay penalties for blocked segments.
  • Control room can instantly see which junctions are isolated.
  • Emergency routes are recalculated around the blockage.
  • Temporary diversions can be planned scientifically.
Pseudocode: checking connectivity with DFS
visited[] = {false}

function dfs(u):
    visited[u] = true
    for each v in adj[u]:
        if not visited[v]:
            dfs(v)

function isNetworkConnected():
    pick any start node s
    dfs(s)
    for each node v:
        if not visited[v]:
            return false
    return true
O(V + E) time Must reset visited[] after each incident

5. Placing Cameras & Sensors with Minimum Cost

Using MST to wire the monitoring network

Traffic sensors and CCTV cameras are usually connected through a communication network. Laying cable for every possible road is too expensive, so we need a minimum cost backbone.

Problem: Connect all camera locations with minimum wiring cost.

Algorithms used
  • Prim's MST – grows the network starting from control room.
  • Kruskal's MST – sorts edges by cost and joins components.

Both algorithms generate a Minimum Spanning Tree (MST), giving a design that covers all points at minimal cable length or cost.

  • Reduced installation and maintenance cost.
  • Every sensor is reachable by the control centre.
  • Monitoring can later be upgraded to smart/AI cameras.
Very short Prim-style pseudocode
function primMST(graph, start):
    for each v:
        inMST[v] = false
        cost[v] = INF
    cost[start] = 0

    repeat (V times):
        u = vertex with min cost[] not in MST
        inMST[u] = true

        for each edge (u, v, w):
            if not inMST[v] and w < cost[v]:
                cost[v] = w
                parent[v] = u

6. Sorting & Analysing Traffic Data

Preparing clean dashboards for decision making

The control room collects gigabytes of data: speed, volume, density, time of day, etc. To show useful charts, data must be sorted and grouped efficiently.

Problem: Sort and filter large logs to find worst hours, roads, and patterns.

Algorithms used
  • Merge Sort – stable sorting for daily traffic records.
  • Quick Sort – fast in-memory sorting for live dashboards.
  • Top 10 most congested roads quickly identified.
  • Peak hour ranges can be visualised for planning.
  • Comparison before/after optimisation is easier.
Array-based Merge Sort (outline)
function mergeSort(a, left, right):
    if left >= right: return

    mid = (left + right) / 2
    mergeSort(a, left, mid)
    mergeSort(a, mid + 1, right)
    merge(a, left, mid, right)
